# Glossary

**Last Updated**: October 2025

Terms and definitions used throughout the Ambé Wellness platform documentation.

---

## Platform Terms

**Ambé Wellness**: The telemedicine and wellness consultation platform

**User**: Patient/client seeking wellness consultations (NOT doctors)

**Doctor**: Healthcare provider offering consultations

**Portal**: Dashboard interface (User Portal or Doctor Portal)

---

## User-Related Terms

**Questionnaire**: Health assessment form completed by users

**Health Field**: Category of health concern (e.g., Women's Health, Heart Health)

**Doctor Matching**: Process of assigning users to appropriate doctors

**Consultation**: Video call appointment between user and doctor

**Report**: Post-consultation summary and recommendations from doctor

**Subscription**: $50/month membership for unlimited messaging and consultations

**Referral Code**: Unique code for users to refer friends (8 characters)

**Referral Credit**: 25% discount earned from successful referrals

---

## Doctor-Related Terms

**Verification**: Admin approval process for doctors

**Schedule**: Doctor's weekly availability for consultations

**Assigned Users**: Patients matched to a specific doctor

**Pending Report**: Completed consultation awaiting doctor's report

**Doctor Notes**: Private notes doctors can add about users

---

## Technical Terms

**Firebase**: Backend service providing authentication and database

**Firestore**: NoSQL database storing all application data

**Firebase Auth**: Authentication service

**Stripe**: Payment processing service

**Agora**: Video consultation service

**Real-time Listener**: Firestore feature for live data updates

**Dual Storage**: Storing data in both user and doctor collections

**Route Group**: Next.js organizational pattern for routes

**Protected Route**: Route requiring authentication

**Server Component**: Next.js component rendered on server

**Client Component**: React component with interactivity

**Middleware**: Code running between request and response

---

## Database Terms

**Collection**: Group of documents in Firestore (like SQL table)

**Document**: Single record in Firestore (like SQL row)

**Subcollection**: Nested collection within a document

**UID**: User ID (unique identifier from Firebase Auth)

**Timestamp**: Firestore timestamp data type

**Query**: Database search/filter operation

**Snapshot**: Point-in-time copy of data

---

## Authentication Terms

**Sign Up**: Creating new account

**Login**: Signing into existing account

**Sign Out**: Logging out of account

**Session**: Period of authenticated access

**Token**: Authentication credential

**User Type**: Either "doctor" or "user"

**Profile**: User/doctor data stored in Firestore

---

## Feature Terms

**Cart**: Shopping cart for products

**Checkout**: Payment and order submission process

**Variant**: Product option (e.g., size, quantity)

**Subscription Discount**: $50 off for active subscribers

**Referral Discount**: 25% off from referral program

**Video Consultation**: Live video call between user and doctor

**Messaging**: Real-time chat between user and assigned doctor

**Appointment**: Scheduled consultation

---

## Development Terms

**Component**: Reusable UI element

**Hook**: React function for state/lifecycle

**Context**: React global state management

**Props**: Component parameters

**State**: Component data that can change

**Effect**: Side effect in React component

**Env Variable**: Environment configuration value

**Build**: Process of creating production version

**Deploy**: Publishing to production

---

## Design Terms

**Design Token**: Predefined design value (color, spacing, etc.)

**Tailwind**: CSS framework used for styling

**Responsive**: Adapts to different screen sizes

**Breakpoint**: Screen width where design changes

**Grid**: Layout system with rows and columns

**Card**: Container component with border/shadow

**Modal**: Overlay dialog window

**Toast**: Brief notification message

---

## Health Terms

**General Health**: Overall wellness concerns

**Women's Health**: Female-specific health issues

**Men's Health**: Male-specific health issues

**Muscular Skeletal**: Bones, muscles, joints

**Heart Health**: Cardiovascular concerns

**Skin & Hair Health**: Dermatological issues

**Mental Emotional Health**: Psychological well-being

**Digestive & Metabolic**: Gut and metabolism

**Oncology**: Cancer-related concerns

**Disabilities**: Physical or cognitive limitations

**Behavioral**: Behavior-related concerns

---

**UX**: User Experience

**DB**: Database
